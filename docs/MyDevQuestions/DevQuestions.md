## 1. C++ 语言问题

### explicit 关键字

- **不懂什么**：为什么需要 `explicit`？隐式转换什么时候会发生？
- **答案**：`explicit` 禁止单参数构造函数的隐式转换
  ```cpp
  class A { explicit A(int x); };
  A a = 5;    // ❌ 报错，不能隐式转换
  A a(5);     // ✅ 可以
  ```

---

### = delete

- **不懂什么**：禁用拷贝为什么有两行？拷贝构造和拷贝赋值有什么区别？
- **答案**：
  | 操作 | 语法 | 场景 |
  |------|------|------|
  | 拷贝构造 | `T b = a;` | 创建新对象时 |
  | 拷贝赋值 | `b = a;` | 已存在对象被赋值时 |
  ```cpp
  class A {
      A(const A&) = delete;           // 禁用 T b = a;
      A& operator=(const A&) = delete; // 禁用 b = a;
  };
  ```

---

### 原子类型

- **不懂什么**：`std::atomic` 为什么不能拷贝？
- **答案**：原子操作在硬件层面保证原子性，拷贝语义未定义

---

### alignas(64)

- **不懂什么**：什么是缓存行？为什么要对齐？
- **答案**：64 字节是 CPU 缓存行大小，对齐避免伪共享

---

### lambda 表达式

- **不懂什么**：lambda 是什么？捕获列表 `[this]` 是什么意思？
- **答案**：lambda 是匿名函数，就地定义就地用
  ```cpp
  // 普通函数
  bool is_empty() { return queue_.empty(); }

  // lambda 版本
  [this] { return queue_.empty(); }
  ```

| 捕获方式 | 含义 |
|----------|------|
| `[]` | 不捕获任何变量 |
| `[this]` | 捕获 this 指针，能用成员变量 |
| `[&]` | 按引用捕获所有外部变量 |
| `[=]` | 按值捕获所有外部变量 |

---

### 条件变量 wait

- **不懂什么**：`cv_.wait(lock, [this] { return queue_.empty(); })` 在干什么？
- **答案**：
  - 条件为 true → 直接返回
  - 条件为 false → 解锁 → 阻塞等待 notify → 加锁 → 重新检查条件

  ```cpp
  // 等价于
  while (!条件) {
      cv_.wait(lock);  // 解锁 → 阻塞 → 加锁 → 继续循环
  }
  ```

## 2. 内存问题

### 内存泄漏

- **不懂什么**：什么是内存泄漏？怎么排查？
- **答案**：分配后未释放，程序越久内存占用越高
- **排查**：`valgrind --leak-check=full ./program`

---

### 野指针

- **不懂什么**：什么是野指针？
- **答案**：指向已释放内存的指针
  ```cpp
  delete ptr;
  cout << ptr->value;  // ❌ 野指针访问
  ```

---

### 循环引用

- **不懂什么**：为什么 shared_ptr 会循环引用导致无法释放？
- **答案**：两个 shared_ptr 互相引用，引用计数永远不为 0

## 3. 多线程问题

### 数据竞争

- **不懂什么**：什么是数据竞争？后果是什么？
- **答案**：多线程同时读/写共享数据，结果不确定
  ```cpp
  // 两个线程同时执行，结果可能是 1 而不是 2
  value++;  // 读→改→写 三步不是原子的
  ```
- **排查**：`g++ -fsanitize=thread -g -O2 source.cpp`

---

### 死锁

- **不懂什么**：死锁是怎么产生的？怎么避免？
- **答案**：互相等待对方持有的锁
  ```cpp
  // 线程A                // 线程B
  lock_a.lock();        lock_b.lock();
  lock_b.lock();        lock_a.lock();  // 死锁！
  ```
- **避免**：固定顺序获取锁，或用 `std::lock` 同时获取

---

### 伪共享

- **不懂什么**：什么是伪共享？为什么用 `alignas(64)`？
- **答案**：同一缓存行的不同变量被多线程修改，导致缓存行频繁失效

## 4. 网络问题

- **不懂什么**：粘包和半包是什么？
- **答案**：

  - 粘包：多个包粘在一起

## 5. 性能问题

### 热点代码

- **不懂什么**：怎么找到热点代码？
- **答案**：`perf record -g && perf report`

---

### 上下文切换

- **不懂什么**：上下文切换为什么影响性能？
- **答案**：切换需要保存/恢复寄存器、TLB、缓存，开销大

### 缓存未命中

- **不懂什么**：CPU 缓存命中率低会怎样？
- **答案**：需要去内存读数据，延迟从 ~1ns 变成 ~100ns

## 6. 日志系统

### 异步写入

- **不懂什么**：什么是异步写入？为什么需要？
- **答案**：

  **同步写入**：必须等日志写入完成才能继续
  **异步写入**：把日志丢进队列就返回，后台线程慢慢写

  | | 同步 | 异步 |
  |---|---|---|
  | 像 | 排队办业务 | 挂号后去喝咖啡等叫号 |
  | 日志时 | `log()` → 等待写入 → 继续 | `log()` → 立即返回 → 继续 |
  | 缺点 | 慢 | 可能丢日志（崩溃时队列里还没写完） |

  异步写入让主线程不管 I/O，只管记录，提高吞吐量。

---

### 日志轮转

- **不懂什么**：日志轮转是什么？为什么需要？
- **答案**：日志文件太大时自动切分

  ```
  app.log          ← 当前日志 (5MB)
  app.log.1        ← 昨天 (20MB)
  app.log.2        ← 前天 (20MB)
  ```

  达到设定大小（如 20MB）时：
  1. `app.log.1` → `app.log.2`
  2. `app.log` → `app.log.1`
  3. 创建新的 `app.log` 继续写

  防止日志撑满磁盘，方便查找特定时间的日志。

---

### 生产者-消费者模式

- **不懂什么**：日志队列怎么工作的？
- **答案**：

  ```cpp
  // 生产者：主线程往队列塞日志
  void push(const string& msg) {
      {
          lock_guard<mutex> lock(mtx_);
          queue_.push(msg);
      }
      cv_.notify_one();  // 通知后台线程
  }

  // 消费者：后台线程从队列取日志写入文件
  string pop() {
      cv_.wait(lock, [this] { return !queue_.empty(); });  // 等队列有数据
      string msg = queue_.front();
      queue_.pop();
      return msg;
  }
  ```

  **常见坑**：
  1. 忘了 `notify`：push 后一定要 notify，否则 pop 会一直卡住
  2. 虚假唤醒：wait 被系统叫醒但条件没满足，lambda 版本自动重新检查条件
