## 哈希槽分片架构

```

│  │   CRC16算法:                                                         │   │
│  │   ┌─────────────────────────────────────────────────────────────┐   │   │

│  │   │  CRC16(key) = 12594 (示例)                                  │   │   │
│  │   │       ↓                                                      │   │   │
│  │   │  slot = 12594 % 16384 = 12594                              │   │   │

│  │   │   SET key value                                             │   │   │
│  │   │        │                                                    │   │   │
│  │   │        ▼                                                    │   │   │
```

## 关键设计点

### 1. 去中心化架构
- 无单一中心节点，所有节点对等
- 通过Gossip协议异步传播状态
- 节点自动发现，动态扩容

### 2. 哈希槽分片
- 16384个槽，均匀分配到主节点
- 客户端直接连接目标节点，无需代理
- 槽迁移时服务不中断

### 3. 主从复制
-异步复制，支持一主多从
- 增量同步 + 全量同步，断线自动重连
- 从节点只读，分担读压力

### 4. 故障转移
- 去中心化的选举机制
- 多数派投票，确保一致性
- 自动故障检测和转移

### 5. 客户端重定向
- MOVED: 永久重定向，更新路由表
- ASK: 临时重定向，不更新路由表
- Smart Client缓存完整槽映射

### 6. 配置一致性
- 每个节点有 currentEpoch，递增
- 配置变更时epoch也递增
- 冲突时epoch大的胜出
