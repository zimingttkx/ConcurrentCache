## 关键设计点

### 1. 多数据类型支持
- 根据数据大小自动选择编码格式
- ziplist: 内存紧凑，适合小数据
- hashtable/skiplist: 支持大数据，O(1)/O(logN)访问

### 2. 混合持久化
- RDB: 适合备份，恢复快，但可能丢失最近数据
- AOF: 适合实时性要求高，最多丢1秒数据
- 两者可以同时启用，互为补充

### 3. 惰性 + 定期过期删除
- 惰性删除: 不浪费CPU，但可能内存泄漏
- 定期删除: 及时回收内存，但消耗CPU
- 两者结合保证内存效率和性能

### 4. 近似LRU/LFU
- 全量精确算法开销大
- 采样近似算法以少量精度换取高性能
- 可通过配置调整精度

### 5. 固定线程池
- 线程数固定，避免线程创建销毁开销
- 绑定CPU核心，减少调度开销
- I/O线程与Worker线程分离，各司其职
