## 关键设计点

### 1. 单Reactor模式
- 所有IO事件在单个EventLoop中处理
- 优点：简单，适合低并发场景
- 缺点：无法利用多核CPU

### 2. 非阻塞IO
- Socket设置为非阻塞模式
- accept/recv/send不阻塞

### 3. RAII资源管理
- Connection析构时自动关闭连接
- 无需手动管理生命周期

### 4. 命令模式
- Command基类定义统一接口
- 新命令只需实现execute()方法
