# 第六章：基础设施层（日志、配置、信号）

## 章节预告

前面几章我们学习了应用层、协议层、存储层、网络层。现在问题是：

**网络层、协议层、应用层都需要输出日志、读取配置、处理信号。这些公共功能应该放在哪里？**

这就是**基础设施层**要解决的问题。

## 6.1 日志系统

### 6.1.1 为什么需要日志？

```
没有日志的问题：

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  程序崩溃了，但你不知道发生了什么                                          │
│                                                                             │
│  $ ./server                                                             │
│  Segmentation fault (core dumped)                                         │
│                                                                             │
│  你只能看到"崩溃了"，但不知道：                                          │
│  - 崩溃在哪里？                                                          │
│  - 崩溃前的操作是什么？                                                 │
│  - 是什么数据导致的崩溃？                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

有日志的好处：

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  $ ./server                                                             │
│                                                                             │
│  [2024-01-15 10:30:15.123] [INFO] Server started on port 6379          │
│  [2024-01-15 10:30:16.456] [INFO] Client connected: fd=5, ip=127.0.0.1   │
│  [2024-01-15 10:30:17.789] [DEBUG] GET request: key=name                 │
│  [2024-01-15 10:30:17.890] [ERROR] RESP parse error: unexpected char    │
│                                                                             │
│  你可以清楚地看到程序运行的每一步，定位问题更容易                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.1.2 日志级别

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           日志级别                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  TRACE (0)   ─ 最详细，开发调试用                                        │

│                                                                             │

│  - 生产环境不需要 TRACE、DEBUG，太多的日志                               │

│  - 提高性能                                                              │
│                                                                             │

```

### 6.1.3 Logger 实现

**文件：** `src/base/log.h`

```cpp
class Logger {
public:
    enum class Level {
        TRACE,
        DEBUG,
        INFO,
        WARN,
        ERROR,
        FATAL
    };

    // 单例访问
    static Logger& instance();

    // 配置
    void setLevel(Level level);
    void setFile(const std::string& filename);

    // 打印日志
    void log(Level level, const char* file, int line, const char* fmt, ...);

private:
    Logger() = default;
    ~Logger();

    Level level_ = Level::INFO;
    FILE* file_ = stdout;  // 默认输出到控制台
};
```

**宏定义简化调用：**

```cpp
// 宏定义，让调用更简洁
#define LOG_TRACE(fmt, ...) \
    Logger::instance().log(Logger::Level::TRACE, __FILE__, __LINE__, fmt, ##__VA_ARGS__)

#define LOG_DEBUG(fmt, ...) \
    Logger::instance().log(Logger::Level::DEBUG, __FILE__, __LINE__, fmt, ##__VA_ARGS__)

#define LOG_INFO(fmt, ...) \
    Logger::instance().log(Logger::Level::INFO, __FILE__, __LINE__, fmt, ##__VA_ARGS__)

#define LOG_WARN(fmt, ...) \
    Logger::instance().log(Logger::Level::WARN, __FILE__, __LINE__, fmt, ##__VA_ARGS__)

#define LOG_ERROR(fmt, ...) \
    Logger::instance().log(Logger::Level::ERROR, __FILE__, __LINE__, fmt, ##__VA_ARGS__)

// 使用示例
LOG_INFO("Server started on port %d", 6379);
LOG_ERROR("Failed to connect: %s", strerror(errno));
```

## 6.3 信号处理

### 6.3.1 什么是信号？

```
信号 = 软件中断，通知进程发生了某个事件

常见信号：

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  SIGINT (Ctrl+C)    ─ 终止进程，正常退出                                │
│  SIGTERM             ─ 终止进程，请求优雅退出                            │
│  SIGSEGV             ─ 段错误，程序崩溃                                  │
│  SIGPIPE             ─ 向已关闭的 socket 写数据                         │
│  SIGHUP              ─ 终端关闭                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.3.2 为什么需要信号处理？

```
没有信号处理的问题：

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  $ ./server                                                             │
│  Server started...                                                       │
│  ...                                                                      │
│  ^C  ← 用户按 Ctrl+C                                                    │
│                                                                             │
│  程序直接退出，可能：                                                     │
│  - 正在写的日志没有刷新                                                   │
│  - 正在处理的请求被中断                                                   │
│  - 客户端连接没有优雅关闭                                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

有信号处理的好处：

┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  $ ./server                                                             │
│  Server started...                                                       │
│  ...                                                                      │
│  ^C                                                                      │
│                                                                             │
│  捕获 SIGINT：                                                           │
│  ┌─────────────────────────────────────────────────────────────────────┐ │
│  │  1. 设置退出标志 quit_ = true                                      │ │
│  │  2. EventLoop 检测到 quit_，退出循环                                 │ │
│  │  3. 等待所有连接处理完成                                            │ │
│  │  4. 刷新日志文件                                                    │ │
│  │  5. 释放资源                                                        │ │
│  │  6. 优雅退出                                                        │ │
│  └─────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
│  "优雅退出" vs "强制退出"                                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.3.3 SignalHandler 实现

```cpp
class SignalHandler {
public:
    // 注册信号处理函数
    static void setup();

private:
    static void handleSignal(int sig);
};
```

**常用信号处理：**

```cpp
void SignalHandler::setup() {
    // SIGINT: Ctrl+C
    signal(SIGINT, handleSignal);

    // SIGTERM: kill 命令默认信号
    signal(SIGTERM, handleSignal);

    // SIGPIPE: 忽略，防止写已关闭 socket 崩溃
    signal(SIGPIPE, SIG_IGN);

    // SIGSEGV: 捕获段错误，打印堆栈
    signal(SIGSEGV, handleSignal);
}

void SignalHandler::handleSignal(int sig) {
    switch (sig) {
        case SIGINT:
        case SIGTERM:
            LOG_INFO("Received signal %d, shutting down...", sig);
            // 设置退出标志，EventLoop 会优雅退出
            EventLoop::getInstance().quit();
            break;

        case SIGSEGV:
            LOG_FATAL("Segmentation fault! Stack trace:");
            // 打印堆栈信息
            printStackTrace();
            abort();  // 崩溃退出
            break;

        default:
            LOG_WARN("Received unexpected signal %d", sig);
            break;
    }
}
```

## 6.4 main.cpp 主函数

### 6.4.1 完整主函数

```cpp
#include "src/network/event_loop.h"
#include "src/network/socket.h"
#include "src/network/connection.h"
#include "src/command/command_factory.h"
#include "src/base/log.h"
#include "src/base/signal.h"

int main() {
    // ===== 1. 初始化日志 =====
    Logger::instance().setLevel(LogLevel::INFO);
    Logger::instance().setFile("logs/server.log");
    LOG_INFO("=== ConcurrentCache Server Starting ===");

    // ===== 2. 设置信号处理 =====
    SignalHandler::setup();

    // ===== 3. 创建事件循环 =====
    EventLoop loop;

    // ===== 4. 创建监听套接字 =====
    Socket server;
    if (!server.bind_and_listen(6379)) {
        LOG_ERROR("Failed to start server on port 6379");
        return 1;
    }

    // ===== 5. 为服务器套接字创建 Channel =====
    Channel server_channel(&loop, server.fd());

    // ===== 6. 设置 accept 回调 =====
    server_channel.set_read_callback([&loop, &server, &server_channel]() {
        // 接受客户端连接
        int client_fd = server.accept();
        if (client_fd >= 0) {
            // 创建新的 Connection
            auto* conn = new Connection(client_fd, &loop);

            // 设置命令处理回调
            conn->set_command_callback([](const RespValue& cmd, Connection* conn) {
                // 提取命令参数
                const auto& arr = cmd.as_array();
                if (arr.empty()) {
                    return;
                }

                std::vector<std::string> args;
                for (size_t i = 0; i < arr.size(); ++i) {
                    args.push_back(arr[i].as_string());
                }

                // 命令名转小写
                if (!args.empty()) {
                    std::string& cmd_name = args[0];
                    std::transform(cmd_name.begin(), cmd_name.end(), cmd_name.begin(),
                                   [](unsigned char c) { return std::tolower(c); });
                }

                // 创建并执行命令
                auto command = CommandFactory::instance().create(args[0]);
                if (command) {
                    std::string response = command->execute(args);
                    conn->send_response(response);
                } else {
                    conn->send_response(RespEncoder::encode_error("ERR unknown command"));
                }
            });
        }
    });

    // ===== 7. 启用读监听 =====
    server_channel.enable_reading();

    LOG_INFO("Server ready, waiting for connections...");

    // ===== 8. 启动事件循环 =====
    loop.loop();

    LOG_INFO("=== ConcurrentCache Server Shutdown ===");
    return 0;
}
```
