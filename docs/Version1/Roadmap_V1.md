## 开发顺序与依赖

```
Week 1: 基础工具
═══════════════════════════════════════════════════════════════════════════

Day 1-2: Logger (日志系统)
  ├─ 输出: spdlog 封装，异步日志
  ├─ 依赖: 无
  └─ 完成标志: 能记录INFO/ERROR日志到文件

Day 3-4: Config (配置解析)
  ├─ 输出: INI/YAML配置文件解析
  ├─ 依赖: Logger
  └─ 完成标志: 能解析端口、日志级别等配置

Day 5: Signal (信号处理)
  ├─ 输出: SIGINT/SIGTERM 处理
  ├─ 依赖: Logger, Config
  └─ 完成标志: Ctrl+C 优雅退出

Day 6-7: Socket + EventLoop (网络基础)
  ├─ 输出: TCP Server，单Reactor
  ├─ 依赖: Logger
  └─ 完成标志: 能接受连接，打印日志


Week 2: 协议与命令
═══════════════════════════════════════════════════════════════════════════

Day 8-9: Buffer + Channel
  ├─ 输出: 可读写缓冲区，事件封装
  ├─ 依赖: Socket, EventLoop
  └─ 完成标志: 数据能缓存和读取

Day 10-11: RESPProtocol (协议解析)
  ├─ 输出: SET/GET/DEL 命令解析
  ├─ 依赖: Connection, Buffer
  └─ 完成标志: 能解析 Redis RESP 协议

Day 12-13: CommandFactory + Commands
  ├─ 输出: GET/SET/DEL 命令执行
  ├─ 依赖: RESPProtocol
  └─ 完成标志: 命令能正确执行


Week 3: 存储与集成
═══════════════════════════════════════════════════════════════════════════

Day 14-15: GlobalStorage (存储层)
  ├─ 输出: unordered_map 存储
  ├─ 依赖: 无
  └─ 完成标志: 数据能存能取

Day 16-17: 连接集成
  ├─ 输出: 完整请求处理流程
  ├─ 依赖: 所有网络层组件
  └─ 完成标志: redis-cli 能连接和操作

Day 18-20: Server Main + 测试
  ├─ 输出: 可运行的服务器
  ├─ 依赖: 所有组件
  └─ 完成标志: SET/GET/DEL 正常工作
```

## 核心技术点

| 组件 | 技术选型 | 理由 |
|------|----------|------|
| 日志 | spdlog | 高性能、异步支持 |
| 网络 | epoll LT | 简单可靠，够用 |
| 协议 | RESP | Redis兼容 |
| 存储 | unordered_map | 最快O(1) |
| 线程 | 无 | 单Reactor单线程 |

- [ ] redis-cli -p 6379 能连接
- [ ] SET key value 返回 OK
- [ ] GET key 返回正确值
- [ ] DEL key 返回删除数量
- [ ] Ctrl+C 优雅退出
- [ ] 无内存泄漏
