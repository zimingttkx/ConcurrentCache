# 高性能分布式内存对象缓存系统（ConcurrentCache）

## 项目简介

ConcurrentCache 是一个从零开始实现的**类 Redis/Memcached 高性能内存对象缓存系统**，专为深入理解网络编程、并发编程、内存管理和缓存系统设计而打造。本项目完美融合了网络编程、线程池、内存池、高并发、缓存替换五大核心技术栈，采用生产级工程化设计，代码结构清晰，可扩展性强。

**项目目标**：
- 彻底吃透计算机网络底层原理和高性能网络编程
- 掌握高并发系统的设计与实现
- 深入理解内存管理和内存池技术
- 学习缓存系统的核心设计思想
- 打造一个可用于生产环境的轻量级缓存系统

## 技术栈

- **编程语言**：C++17（使用现代C++特性，如智能指针、lambda表达式、std::thread等）
- **操作系统**：Linux（Ubuntu 20.04+ / CentOS 8+）
- **网络模型**：主从Reactor多线程模型（基于Linux epoll边缘触发）
- **内存管理**：三级分层内存池（对标tcmalloc）
- **并发控制**：分段锁、读写锁、自旋锁、原子操作
- **缓存替换**：LRU、LFU、FIFO、Random
- **协议支持**：Redis RESP协议（兼容所有Redis客户端）
- **持久化**：RDB快照、AOF追加日志
- **分布式**：Redis Cluster风格哈希槽分片、主从复制、故障转移
- **依赖库**：
  - spdlog：高性能日志库
  - gtest：单元测试框架
  - google-benchmark：性能测试框架

## 核心特性

### 已规划特性

- 主从Reactor高并发网络模型，支持万级并发连接
- 三级分层高并发内存池，减少内存碎片和锁竞争
- 固定任务线程池，解耦I/O与业务逻辑
- 分段锁全局哈希表，O(1)时间复杂度的KV操作
- 多策略缓存替换算法（LRU/LFU/FIFO/Random）
- 键过期自动删除（惰性删除+定期删除）
- 支持Redis RESP协议，兼容所有Redis客户端
- 支持5种基本数据类型：字符串(String)、哈希(Hash)、列表(List)、集合(Set)、有序集合(ZSet)
- RDB快照持久化和AOF追加日志持久化
- 集群模式：哈希槽分片、主从复制、故障转移

### 未来扩展特性

- 支持Lua脚本
- 支持发布订阅(Pub/Sub)
- 支持事务
- 支持流数据(Stream)
- 支持地理空间(Geo)

## 快速开始

### 环境准备

```bash
# 安装依赖
sudo apt update
sudo apt install -y build-essential cmake libspdlog-dev libgtest-dev libbenchmark-dev

# 克隆项目
git clone https://github.com/your-username/ConcurrentCache.git
cd ConcurrentCache
```

### 编译项目

```bash
mkdir build && cd build
cmake ..
make -j$(nproc)
```

### 运行服务器

```bash
./bin/concurrentcache-server --config ../conf/concurrentcache.conf
```

### 使用客户端连接

```bash
# 使用redis-cli连接
redis-cli -p 6379

# 测试命令
127.0.0.1:6379> SET key1 value1
OK
127.0.0.1:6379> GET key1
"value1"
127.0.0.1:6379> EXPIRE key1 10
(integer) 1
127.0.0.1:6379> TTL key1
(integer) 8
```

## 开发指南

### 代码规范

- 严格遵循Google C++代码规范
- 使用4个空格缩进，禁止使用Tab
- 变量名：小写下划线命名法（snake_case）
- 类名/结构体名：大驼峰命名法（CamelCase）
- 常量名：全大写下划线命名法（UPPER_SNAKE_CASE）
- 函数名：小写下划线命名法
- 每个文件开头添加版权声明和文件说明
- 关键函数和类添加详细的Doxygen风格注释

### 提交规范

- 提交信息格式：`type(scope): message`
- type：feat（新功能）、fix（修复bug）、docs（文档）、style（代码格式）、refactor（重构）、test（测试）、chore（构建工具或辅助工具的变动）
- scope：修改的模块，如base、network、cache等
- message：简短描述修改内容，不超过50个字符

### 测试规范

- 每个模块都要有对应的单元测试
- 单元测试覆盖率不低于80%
- 关键功能要有集成测试
- 每次提交前必须运行所有测试，确保全部通过

## 性能指标

### 单实例性能目标

- QPS：> 100,000（GET/SET混合操作，比例1:1）
- 平均延迟：< 1ms
- P99延迟：< 5ms
- 最大并发连接数：> 10,000
- 内存使用率：> 90%（有效数据占比）

### 集群性能目标

- 线性扩展：性能随节点数增加近似线性增长
- 故障转移时间：< 30s
